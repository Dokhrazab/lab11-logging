# lab11-logging
Энэхүү лабораториор Java программд Log4j 2 санг ашиглан хэрхэн лог хөтлөх, гарсан алдааг (debug) терминалын командуудаар хэрхэн шинжлэх талаар суралцлаа.

1. Ангиудын тайлбар
BankAccount: Банкны дансны үйлдэл буюу мөнгө хийх (deposit), авах (withdraw) үйлдлүүдийг гүйцэтгэнэ. Үйлдэл бүрийг TRACE, DEBUG, INFO, WARN, ERROR, FATAL гэсэн 6 түвшний лог ашиглан хянадаг. 
+1

Customer: Хэрэглэгчийн нэр, имэйлийг хадгална. getDomain() функц нь имэйлээс домэйныг (жишээ нь: @gmail.com) салгаж авдаг. 

2. Log4j2.xml тохиргоо
Лог хөтлөхдөө дараах хоёр хэлбэрийг ашигласан: 

Console Appender: Командын мөрөнд (Terminal) логийг цаг хугацаа, thread ID, түвшинтэй нь харуулна. 
+1

RollingFile Appender: Логуудыг logs/app.log файлд хадгална. 
+1

Файл солих (Rotation): SizeBasedTriggeringPolicy ашиглан лог файл 1MB хүрэх бүрт шинэ файл үүсгэж, хуучныг нь архивлах (gz) тохиргоо хийсэн. 

3. Даалгаврын хариулт
Алхам 2.1: Лог түвшний шалгалт

Main классаас дансны үйлдлүүдийг (хэвийн, сөрөг утгатай, үлдэгдэл хүрэхгүй гэх мэт) 4-өөс олон удаа дуудаж туршсан. Үр дүнд нь app.log файл дээр бүх 6 түвшний лог амжилттай бичигдсэн. 
+1

Алхам 4.1: tail -f vs cat/less

tail -f команд нь файлыг нээлттэй хэвээр үлдээж, шинээр нэмэгдэж буй лог бүрийг real-time буюу шууд харуулдаг. Харин cat эсвэл less нь тухайн үеийн статик мэдээллийг л харуулдаг тул программ ажиллаж байх явцад алдааг шууд барьж авахад tail -f илүү тохиромжтой. 
+1

4. Алдааг илрүүлж зассан түүх
Customer ангийн getDomain() функц дээр имэйл хаяг нь null байх үед NullPointerException үүсэж байсан. 
+1

Илрүүлсэн арга:

Лог дээр logger.debug("getDomain called with email={}", email); гэж бичиж шалгахад email хувьсагч хоосон ирж байгааг мэдсэн. 
+1

Зассан арга:

Имэйл null байгаа эсэхийг шалгадаг нөхцөл (if check) нэмж, хэрэв хоосон бол WARN лог үлдээгээд "UNKNOWN" гэж буцаадаг болгож зассан. Мөн хувийн мэдээллийг хамгаалах үүднээс имэйлийг маскалж (masking) харуулдаг болгож сайжруулав. 
+2
